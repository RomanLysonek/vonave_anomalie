NOTINOANOMALY RESEARCH

Knowledge transfer into demand forecasting

Evaluation

Did anomaly-aware forecasting help?

Every anomaly intervention had to improve the same chronological forecasting objective as the untouched control. Interesting anomaly rankings alone were never sufficient.

Leakage-safe experiment contract

The detector and forecast are rebuilt as they would have existed at each historical origin.

01 / Fit

Past only

Expectations, robust scaling, EVT thresholds and optional autoencoder preprocessing are fitted on the fold's training history.

No future-derived medians
02 / Expose

Origin state

Only anomaly state available by the forecast origin may enter a feature. Target-row scores can affect historical loss weights, not predictors.

Feature/loss paths separated
03 / Compare

Same rows

Control and intervention are scored on aligned product-dates under the same rolling origins, metric population and forecast objective.

No convenient exclusions
04 / Decide

Bounded promotion

Development evidence selects; recent benchmark and frozen audit only confirm. Unsupported or contaminated artifacts cannot promote a policy.

Control is the default

Interventions tested

All actions were explicit, reversible and compared with anomaly_mode=off.

Features

Origin-known anomaly state

Recent exceedance rate, severity history and days since an exceedance test whether regime instability helps the forecast.

Weights

Bounded robust fitting

Likely idiosyncratic historical labels can receive less influence, while minimum weights protect event spikes and systemic shocks.

Both

Combined policy

Feature and loss paths can be enabled together, but their interaction must beat the identical off-mode control.

Specialists

Gates and blends

Anomaly-aware and recent-regime specialists were framed as bounded candidates; historical preflight evidence is excluded from selection.

Evidence accepted

Statistical auditDeterministic, checked-in causal score and context-novelty artifacts with source fingerprints.
Weight ablationThe verified comparison selected control; weighting did not improve the development or recent benchmark objective.
V2 only when verifiedAutoencoder evidence remains unavailable unless schema, canonical inputs, outputs and provenance fingerprints validate.

Evidence excluded

Legacy compact autoencoderExcluded because preprocessing used future-derived medians.
Historical searchesContaminated benchmark evidence and unavailable result trees are archived, not inferred.
Weekend-v2 preflightProvenance-unverified and retained only as motivation; no weekend-v2 or final execution was run.
Decision

The negative result is the result

The transferred anomaly methods produce useful diagnostics and a defensible review queue, but the verified forecasting evidence does not justify changing the submitted forecast. Control / anomaly_mode=off remains recommended.

No precision, recall or false-alarm rate is reported because no adjudicated anomaly labels exist.
